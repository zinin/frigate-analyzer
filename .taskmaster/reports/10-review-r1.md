**[IMPORTANT] QuickExportHandlerTest.kt:289–298 — Indirect "video was sent" assertion is fragile and may produce false positives**
The happy-path test detects video sending by checking `capturedRequests.size > knownRequestTypes` (where known types are `AnswerCallbackQuery`, `EditChatMessageReplyMarkup`, `SendTextMessage`). This confirms *something unexpected* was sent, not that a `SendVideo` request was sent. If the handler adds any other bot interaction in the future (e.g., `DeleteMessage`), the assertion would pass vacuously. This could mask regressions where video sending is broken but another request type is introduced.
*Suggested fix*: Check the concrete type via `filterIsInstance<SendVideo>()` (or the actual tgbotapi multipart-upload request class). If the type is difficult to express due to tgbotapi's extension function design, add a comment acknowledging the limitation and documenting the expected request type name.
