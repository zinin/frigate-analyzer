# Merged Design Review — Iteration 1

## codex-executor (gpt-5.3-codex)

### DESIGN-1: Отсутствие обработки ошибок при вызове setMyCommands

**Серьёзность:** Высокая
**Файл:** `FrigateAnalyzerBot.kt`
**Описание:** Ни в дизайне, ни в плане реализации не предусмотрена обработка ошибок при вызове `setMyCommands`. Если Telegram API вернет ошибку (например, при сетевых проблемах или rate limiting), исключение может прервать весь процесс запуска бота в методе `start()`. Поскольку `registerDefaultCommands()` вызывается в `start()` после `bot.getMe()`, но до `buildBehaviourWithLongPolling`, необработанное исключение остановит запуск long polling, и бот вообще не будет работать.
**Рекомендация:** Обернуть вызовы `setMyCommands` в `try-catch` блоки.

### DESIGN-2: Дублирование списка команд

**Серьёзность:** Средняя
**Файл:** `FrigateAnalyzerBot.kt`
**Описание:** Команды `/start` и `/help` дублируются в обоих методах — `registerDefaultCommands()` и `registerOwnerCommands()`. При добавлении новых команд или изменении описаний потребуется менять код в двух местах.
**Рекомендация:** Вынести общие команды в константу и строить список для owner через объединение.

### PLAN-1: Некорректная сигнатура setMyCommands с vararg и именованным параметром scope

**Серьёзность:** Средняя
**Файл:** `FrigateAnalyzerBot.kt`
**Описание:** Нужно проверить, что ktgbotapi 30.0.2 предоставляет vararg-версию `setMyCommands`. В некоторых версиях библиотеки может использоваться `List<BotCommand>` вместо `vararg`.
**Рекомендация:** Проверить актуальную сигнатуру. API Reference указывает `vararg`, но стоит верифицировать.

### DESIGN-3: Гонка между startup и /start для owner-команд

**Серьёзность:** Низкая
**Описание:** Два одновременных вызова `setMyCommands` для одного scope теоретически безопасны, но создают лишнюю нагрузку.
**Рекомендация:** Никаких изменений не требуется — крайне маловероятный сценарий.

### DESIGN-4: Не рассмотрен сценарий удаления команд при деактивации

**Серьёзность:** Низкая
**Описание:** Если owner будет удален, его расширенное меню останется до перезапуска бота.
**Рекомендация:** Зафиксировать как известное ограничение.

### PLAN-2: Не указано точное место вставки в handleStart()

**Серьёзность:** Низкая
**Описание:** План не уточняет, что `registerOwnerCommands(chatId)` должен быть только внутри ветки `if (username == properties.owner)`.
**Рекомендация:** Явно указать место вставки.
