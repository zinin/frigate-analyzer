# Quick Export: вторая кнопка с аннотациями + прогресс

## Контекст

Сейчас при обнаружении объекта в уведомлении есть одна кнопка "📹 Экспорт видео", которая экспортирует оригинальное видео без аннотаций (`ExportMode.ORIGINAL`). Команда `/export` позволяет выбрать режим (оригинал / с объектами), но требует 4 шага диалога.

## Цель

Добавить вторую кнопку быстрого экспорта с аннотациями и отображение прогресса для обеих кнопок.

## Дизайн

### Inline-клавиатура

**Было:**
```
[ 📹 Экспорт видео ]
```

**Стало:**
```
[ 📹 Оригинал ] [ 📹 С объектами ]
```

Callback data:
- `qe:{UUID}` — оригинал (без изменений)
- `qea:{UUID}` — с аннотациями (новый префикс)

При нажатии любой кнопки обе заменяются на одну кнопку прогресса. После завершения/ошибки клавиатура восстанавливается к двум кнопкам.

### Обработка callback

Роутинг в `FrigateAnalyzerBot`: добавить фильтр по префиксу `qea:` и направлять в `QuickExportHandler`, либо расширить существующий фильтр для обоих префиксов.

`QuickExportHandler.handle()` определяет `ExportMode` по префиксу callback data:
- `qe:` → `ExportMode.ORIGINAL`
- `qea:` → `ExportMode.ANNOTATED`

### Прогресс в кнопке

Вместо статичной "⚙️ Экспорт..." — обновление текста кнопки по этапам:
- `⚙️ Склейка видео...`
- `⚙️ Сжатие видео...`
- `⚙️ Аннотация 45%...`
- `⚙️ Отправка...`

Одна кнопка прогресса (при повторном нажатии — "Экспорт уже выполняется"). Обновление по смене этапа и по порогу ≥5% для ANNOTATING (аналогично `/export` ExportExecutor).

Прогресс отображается для обоих режимов (оригинал и аннотированный).

### Таймауты

- `ExportMode.ORIGINAL`: 5 минут (300 000 мс) — без изменений
- `ExportMode.ANNOTATED`: 20 минут (1 200 000 мс) — аналогично `/export`

### Изменения в сервисном слое

`VideoExportService.exportByRecordingId()` — добавить параметр `mode: ExportMode = ExportMode.ORIGINAL` и пробросить в `exportVideo()`. Остальная логика (merging, compressing, annotating) уже реализована.

### Обработка ошибок

- **Конкурентность:** существующий `activeExports: MutableSet<UUID>` защищает от дублирования — не зависит от режима
- **Таймаут:** сообщение об ошибке + восстановление клавиатуры с двумя кнопками
- **Rate limiting:** обновление по смене этапа + порог ≥5% для ANNOTATING (аналогично ExportExecutor)
